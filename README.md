# Clinic Translator

A HIPAA-compliant real-time Spanish-English translator built for single-clinic phone triage. When a Spanish-speaking patient calls in, staff use this app to read a live English transcript of what the patient is saying, type a reply in English, and have that reply synthesized as natural-sounding Spanish and played back through the handset — all in under two seconds.

Staff are always in control. No automated voice agent. No auto-send. Every message goes out only when staff approves it.

Full design specification: [`docs/superpowers/specs/2026-05-20-clinic-translator-design.md`](docs/superpowers/specs/2026-05-20-clinic-translator-design.md)

---

## How it works

A call arrives. Staff click **Start Call**. The browser requests microphone access and opens a WebSocket to Deepgram Nova-3, which streams Spanish speech-to-text in real time. Partial words appear greyed out as the patient speaks; finalized utterances lock in and are immediately sent through the translation pipeline. The English translation renders beneath each Spanish line with any recognized medical terms highlighted from the clinic's glossary.

When staff are ready to respond, they type in English and press Cmd+Enter. The app translates the reply to Spanish, hands it to Deepgram Aura-2 TTS, and plays the audio through the default output device. Staff hear what the patient hears before it goes out. Common follow-up replies can be generated by the AI suggestion engine (Track C1), which streams a short draft based on conversation history — staff can accept, edit, or dismiss it. Suggestions never send automatically.

Urgency detection runs continuously. If the patient says anything matching the emergency keyword set (cardiac, respiratory, bleeding, obstetric, trauma), the call is automatically flagged urgent, a soft audio alert plays, and the AI suggests escalation language.

## Database schema

Seven tables. All patient-identifiable data (names, phone numbers, transcripts, translations, notes) is encrypted at the application layer with AES-256-GCM before hitting Postgres.

**`patients`** — Patient records. Encrypted: name, callback phone, DOB last 4, notes.

**`calls`** — One row per phone call. Tracks staff member, urgency level (low / normal / high / urgent), and outcome (completed / transferred / voicemail / dropped / fallback).

**`utterances`** — Every spoken or typed message in the call. Stores encrypted ES text, encrypted EN translation, and role (patient / staff). AI suggestion columns live here too: suggestion text, confidence score, outcome (accepted / edited / dismissed), and escalate flag.

**`staff_users`** — Clinic staff accounts with role (owner / staff / admin).

**`glossary_terms`** — Per-clinic medical term overrides. Supports dialect tagging (Mexican / Central American / Caribbean) and category (drug / procedure / intake / billing / scheduling).

**`clinic_settings`** — Single row per clinic. Stores provider configs (JSONB) for STT, translate, TTS, and suggest; latency mode; dialect preference; clinic name, hours, services, FAQs, transfer phone, and escalation rules.

**`audit_log`** — Immutable. Records every read and write of patient/call/utterance data, auth events, and decryption failures. UPDATE and DELETE are revoked at the database level.

## Call data flow

**Patient speaking (Spanish to English)**

1. Staff clicks Start Call — `POST /api/sessions` creates a call record
2. Browser opens mic at 16kHz mono, sends 100ms PCM chunks via WebSocket to `/api/stt`
3. Deepgram Nova-3 returns partials (greyed) and finals (locked)
4. Finals hit `POST /api/translate` — Groq Llama translates, glossary terms are injected
5. EN translation renders in PatientPane beneath the ES source
6. Utterance written to DB (both texts encrypted), audit log entry written

**Staff replying (English to Spanish)**

1. Staff types English reply, presses Cmd+Enter
2. `POST /api/translate {src:'en', dst:'es'}` — ES preview rendered
3. `POST /api/tts {text, voice:'Achernar'}` — Deepgram Aura-2 synthesizes MP3
4. Cache checked by `hash(text + voice)` — hit returns instantly, miss writes cache
5. MP3 plays through AudioContext → default audio output
6. Both EN and ES utterances written to DB (encrypted), audit log entry written

**AI suggestions (Track C1)**

1. After each patient utterance, `POST /api/suggest` streams an SSE response
2. Groq Llama generates a ≤2-sentence English reply draft with confidence score and escalate flag
3. Draft appears as ghost text in StaffPane
4. Staff accepts, edits, or dismisses — outcome recorded in `utterances.suggestion_outcome`
5. Suggestion never sends without staff action

## Security and HIPAA compliance

**Encryption** — AES-256-GCM with a 32-byte key from `PHI_ENCRYPTION_KEY`. Format: `IV (12 bytes) || ciphertext || auth tag (16 bytes)`. Applied to every field that could identify a patient before any DB write.

**Audit trail** — Every access and modification is logged to `audit_log` with actor, action, target, timestamp, and IP. The table has UPDATE and DELETE revoked so the trail cannot be altered after the fact.

**PHI log prevention** — A custom ESLint rule (`eslint-rules/no-phi-log.js`) blocks `console.log` calls that reference field names: `text`, `translation`, `name`, `phone`, `dob`. Enforced at lint time.

**Sentry scrubbing** — The `beforeSend` hook strips any PHI-like fields before events leave the server. Sentry is opt-in via `SENTRY_DSN`.

**Access control** — All `/app/*` and `/settings/*` routes protected by NextAuth middleware. Staff must be on the clinic's email allowlist (`CLINIC_EMAIL_ALLOWLIST`).

## Provider system

The provider registry (`lib/providers/registry.ts`) decouples the app from any single vendor. Each category (STT, translate, TTS, suggest) lists available vendors, and the active selection is stored per-clinic in `clinic_settings`. Swapping a provider is a settings change, not a code change.

**Latency presets** control the quality-cost tradeoff:

- **Fast** — Deepgram Nova-3, Groq Llama 3.1 8B, Deepgram Aura-2. Lowest latency.
- **Balanced** (default) — Deepgram Nova-3, Groq Llama 3.3 70B, Deepgram Aura-2.
- **Accurate** — Same stack with higher temperature / beam settings for difficult audio.

Unimplemented vendor stubs (Google Speech, Azure OpenAI, DeepL, Cartesia, ElevenLabs, Polly) throw `ProviderNotImplementedError`. Stubs exist so they appear in the admin UI and can be fully wired without touching the rest of the codebase.

## Real-time edge processors

These run client-side on the live audio and transcript stream:

| Processor | Trigger | Effect |
|---|---|---|
| `urgency-keywords` | ~40 emergency phrases in ES | Elevates urgency to "urgent", plays alert tone, suggests transfer |
| `silence-detector` | No audio for 8 seconds | Prompts staff to send a Spanish check-in |
| `hangup-detect` | No activity for 3 minutes | Auto-ends the call |
| `callback-verify` | Phone number pattern in ES transcript | Surfaces a card for staff to confirm the number |
| `multi-speaker` | Inter-utterance pause pattern | Flags possible second voice on patient's line |
| `correction` | Right-click on any transcript token | Inline edit popover, updates translation |
| `code-switch` | Mixed ES/EN in utterance | Flags for re-translation |

## Getting started

```sh
git clone <repo-url> clinic-translator
cd clinic-translator
cp .env.example .env.local
npm install
npm run db:migrate
npm run dev
# http://localhost:3000
```

For local development without Google OAuth, use the dev login bypass at `/api/dev-login`.

**Required environment variables**

```
# Auth
NEXTAUTH_SECRET=          # any random string
NEXTAUTH_URL=http://localhost:3000
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
CLINIC_EMAIL_ALLOWLIST=   # comma-separated, supports *@clinic.com wildcards

# Voice APIs
DEEPGRAM_API_KEY=         # STT (Nova-3) + TTS (Aura-2)
GROQ_API_KEY=             # Translation + AI suggestions

# PHI encryption
PHI_ENCRYPTION_KEY=       # base64-encoded 32 random bytes

# Database
DATABASE_URL=             # Neon Postgres connection string

# Optional
SENTRY_DSN=
```

Generate `PHI_ENCRYPTION_KEY`:

```sh
openssl rand -base64 32
```

## Testing

**Unit tests** (`tests/unit/`) cover crypto round-trips and key rotation, medical glossary substitution with dialect priority, Zustand session store (append, resume, partial overwrite), urgency keyword detection and audio alerts, silence detection, correction popover behavior, and provider registry dispatch.

**Integration tests** (`tests/integration/`) hit the actual API routes against a test database: translation with glossary injection, TTS cache hit on repeated input, auth allowlist enforcement, and encryption-at-rest verified by querying Postgres directly.

**E2E tests** (`tests/e2e/happy-path.spec.ts`) use Playwright with a fake audio device. Scenarios include the full call round-trip (mic → ES transcript → EN translation → typed EN → ES audio plays), Deepgram WebSocket disconnect mid-stream with no transcript loss, translate API 500 with retry button, and browser refresh mid-call with session recovery from IndexedDB.

## Deployment

Hosted on Vercel with Neon Postgres as the database. Vercel Enterprise is required for HIPAA BAA. STT WebSocket proxy runs on Vercel Edge; translation, TTS, and suggest routes run on Node.js runtime.

Estimated operating cost at 50 calls per day: Vercel ~$300/mo, Deepgram ~$120/mo, Groq ~$30/mo, Neon ~$50/mo, Sentry ~$80/mo. Free-tier configuration (Neon free, Deepgram growth tier) is viable for a pilot.
